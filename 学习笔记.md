## 一、什么是 JWT？

JWT（JSON Web Token）是一种用于身份认证的开放标准。它由三部分组成，用 `.` 分隔：

```
eyJhbGciOi...  .  eyJ0b2tlbl...  .  QJR-vICab...
    HEADER         PAYLOAD         SIGNATURE
```

| 部分 | 内容 | 说明 |
|------|------|------|
| **Header** | `{"alg":"HS256", "typ":"JWT"}` | 声明签名算法 |
| **Payload** | `{"user_id":1, "exp":1779099253, ...}` | 存用户信息，**但不要存密码！** Payload 是明文 base64，任何人都能解开看 |
| **Signature** | 对前两部分的哈希签名 | 用服务器密钥加密，防篡改。篡改后签名对不上，服务器立刻拒绝 |

## 二、为什么需要两种 Token？

本项目 SimpleJWT 配置了两种 token：

| Token 类型 | 有效期 | 作用 |
|-----------|--------|------|
| **access token** | 2 小时 | 每次访问 API 都要带着，证明"我是谁" |
| **refresh token** | 7 天 | access 过期后，用它换新的 access，不用重新输密码 |

### 为什么要分两个？

这是一个经典的安全设计模式：

```
access 过期快 → 即使被黑客截获，只有 2 小时攻击窗口
refresh 存得久 → 但它几乎不用（只在续期时发送一次），暴露风险极低
```

可以这样类比：

> **access token** = 临时工牌，每天上班去前台换新的，掉了也没事反正当天就作废  
> **refresh token** = 入职证明，锁在抽屉里，只有换工牌的时候才拿出来一次

## 三、完整登录流程

### 3.1 登录 — POST /api/token/

```
┌──────────────────────────────────────────────────────────────┐
│  前端（Vue）                    后端（Django）                  │
│                                                                │
│  POST /api/token/                                              │
│  Body: {                                                       │
│    "username": "admin",         ───→  1. 查数据库，验证密码      │
│    "password": "12138"                2. 密码正确？用密钥签名     │
│  }                                    3. 生成 access + refresh  │
│                                                                │
│  响应: {                              ←─── 4. 返回两个 token      │
│    "access":  "eyJ...",                                        │
│    "refresh": "eyJ..."                                         │
│  }                                                             │
│                                                                │
│  5. 前端把两个 token 存到 localStorage                         │
└──────────────────────────────────────────────────────────────┘
```

实际测试结果（用 admin/12138 测的）：

```json
{
    "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**接口只接受 POST 方法**。在浏览器地址栏直接访问会报 `405 Method Not Allowed`。

### 3.2 访问受保护资源 — 带 access token 发请求

```
┌──────────────────────────────────────────────────────────────┐
│  前端（Vue）                    后端（Django）                  │
│                                                                │
│  GET /api/some-protected-api/                                  │
│  Headers: {                                                    │
│    Authorization: Bearer eyJ...   ───→  1. 取出 Header 中的 token
│  }                                      2. 用密钥验证签名的有效性
│                                         3. 解析出 user_id        │
│                                         4. 从数据库查用户信息    │
│                                                                │
│  响应: {                              ←─── 5. 返回用户专属数据    │
│    "id": 1,                                                   │
│    "username": "admin",                                        │
│    "data": "..."                                              │
│  }                                                             │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 Token 续期 — POST /api/token/refresh/

当 access token 过期（2 小时后），不用重新输密码：

```
┌──────────────────────────────────────────────────────────────┐
│                                                                │
│  POST /api/token/refresh/                                      │
│  Body: {                                                       │
│    "refresh": "eyJ..."           ───→  1. 验证 refresh 是否有效  │
│  }                                    2. 有效？颁发新 access     │
│                                                                │
│  响应: {                              ←─── 3. 返回新 access      │
│    "access": "eyJ(new)..."                                    │
│  }                                                             │
│                                                                │
└──────────────────────────────────────────────────────────────┘
```

## 四、相关配置（settings.py）

### 4.1 REST Framework 认证引擎

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': (
        'rest_framework_simplejwt.authentication.JWTAuthentication',
    ),
}
```

这句话的意思是：**所有 DRF 接口默认都用 JWT 认证**。如果某个接口不需要认证，可以单独在视图里覆盖。

### 4.2 SimpleJWT 详细参数

```python
SIMPLE_JWT = {
    'ACCESS_TOKEN_LIFETIME': timedelta(hours=2),     # access 有效期 2 小时
    'REFRESH_TOKEN_LIFETIME': timedelta(days=7),     # refresh 有效期 7 天

    'ROTATE_REFRESH_TOKENS': True,       # 刷新 access 时，同时换一个新的 refresh
    'BLACKLIST_AFTER_ROTATION': True,    # 旧的 refresh 加入黑名单，不能再用了

    'AUTH_HEADER_TYPES': ('Bearer',),    # 请求头格式：Authorization: Bearer <token>
}
```

| 参数 | 解释 |
|------|------|
| `ROTATE_REFRESH_TOKENS` | True = 每次刷新 access 时，也发一个新的 refresh。这样即使旧的 access 和 refresh 都被截获，攻击者也只能用一次 |
| `BLACKLIST_AFTER_ROTATION` | 被替换掉的旧 refresh 直接作废，防止重放攻击 |
| `AUTH_HEADER_TYPES` | 前端请求头里写成 `Bearer xxx`，这是行业标准写法 |

### 4.3 URL 路由配置

```python
# web/urls.py
from rest_framework_simplejwt.views import TokenObtainPairView, TokenRefreshView

urlpatterns = [
    path('api/token/', TokenObtainPairView.as_view(), name='token_obtain_pair'),
    path('api/token/refresh/', TokenRefreshView.as_view(), name='token_refresh'),
]
```

这是 SimpleJWT 内置的两个视图：
- `TokenObtainPairView` → 接收 username + password，返回 access + refresh
- `TokenRefreshView` → 接收 refresh，返回新 access

我们完全不用自己写登录逻辑，SimpleJWT 已经封装好了。

## 五、Token 内部长什么样（解码示例）

用刚才测试拿到的 token 解码 payload：

**access token 的 payload：**
```json
{
    "token_type": "access",
    "exp": 1779099253,          // 过期时间戳（2小时后）
    "iat": 1779092053,          // 签发时间戳（当前时间）
    "jti": "8c672e66...",       // token 唯一 ID
    "user_id": 1                // 用户 ID = admin
}
```

**refresh token 的 payload：**
```json
{
    "token_type": "refresh",
    "exp": 1779696853,          // 过期时间戳（7天后）
    "iat": 1779092053,
    "jti": "95d5cd8f...",
    "user_id": 1
}
```

两个 token 的区别只在 `token_type` 和 `exp`（过期时间）不同。

## 六、前端怎么用？（Vue 3 示例思路）

```javascript
// 1. 登录
const res = await axios.post('http://127.0.0.1:8000/api/token/', {
  username: 'admin',
  password: '12138'
})
localStorage.setItem('access', res.data.access)
localStorage.setItem('refresh', res.data.refresh)

// 2. 访问需要认证的 API
const res2 = await axios.get('http://127.0.0.1:8000/api/some-api/', {
  headers: {
    Authorization: `Bearer ${localStorage.getItem('access')}`
  }
})

// 3. token 过期后刷新
const res3 = await axios.post('http://127.0.0.1:8000/api/token/refresh/', {
  refresh: localStorage.getItem('refresh')
})
localStorage.setItem('access', res3.data.access)
```

## 七、常见疑问

### Q: 为什么浏览器直接打开 /api/token/ 报 405？
> 因为它是 POST 接口，浏览器地址栏发的是 GET 请求。前端用 JavaScript 发 POST 才能正常调用。

### Q: 为什么 token 存 localStorage 而不是 cookie？
> localStorage 不受 CSRF 攻击影响。如果用 cookie，每次请求会自动带上，容易被伪造表单提交攻击。

### Q: password 会被存在 token 里吗？
> **不会。** Token 的 payload 只存 `user_id`，不存密码。密码只在登录那一次用到，验证完就丢了。

### Q: 用户改了密码，旧的 token 还能用吗？
> 默认情况下 **能**。因为 JWT 是无状态的，服务器不存 token，只验证签名。如果需要"改密码就作废旧 token"，需要额外实现（可以用 token 黑名单或版本号机制）。
