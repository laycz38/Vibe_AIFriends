## 一、项目概述

AIFriends 是一个 AI 面经社区平台，前后端分离架构：
- **后端**：Django 6.0.5（运行在 `http://127.0.0.1:8000`）
- **前端**：Vue 3 + Vite + TailwindCSS v4 + daisyUI v5（运行在 `http://localhost:5173`）
- 前后端域名不同，已配置 CORS 跨域支持
- 用户认证：JWT（`djangorestframework-simplejwt`），access_token 2h + refresh_token 7d
- 页面风格：daisyUI 组件驱动 + Tailwind utility class
- 核心功能：用户注册登录、发布面经、搜索浏览、点赞评论、个人资料管理

## 三、前端布局体系

### 3.1 NavBar 全局布局（核心）

[NavBar.vue](file:///d:/vibe_coding/AIFriends/frontend/src/components/navbar/NavBar.vue) 使用 daisyUI `drawer` 组件实现响应式左右分栏：

```
drawer lg:drawer-open
├── drawer-content
│   ├── navbar (bg-base-100 shadow-sm)
│   │   ├── navbar-start: 汉堡菜单 + "AI 面经"
│   │   ├── navbar-center: 搜索框 (join: input + button)
│   │   └── navbar-end: 创作按钮 + UserMenu / 登录
│   └── <slot /> → RouterView
└── drawer-side
    └── menu (bg-base-200)
        ├── 首页 (HomepageIcon)
        ├── AI 问答 (FriendIcon)
        ├── 发布面经 (CreateIcon)
        └── 我的收藏 (StarIcon)
```

关键特性：
- **lg:drawer-open** — 大屏幕侧栏常开（`is-drawer-open:w-54` 折叠 `is-drawer-close:w-16`）
- 折叠时菜单项显示 **tooltip** (`is-drawer-close:tooltip-right`)
- 搜索框 `watch route.query.q` 双向同步 + `handleSearch()` 跳转 `/?q=xxx`
- 图标全部为独立 SVG SFC，使用 `size-6` Tailwind 尺寸类

### 3.2 首页卡片网格

[FeedWaterfall.vue](file:///d:/vibe_coding/AIFriends/frontend/src/components/FeedWaterfall.vue)：

```
grid grid-cols-[repeat(auto-fill,minmax(240px,1fr))] gap-9 mt-12 px-9
```

- 最小列宽 240px，自动填充，间距 36px
- `watch route.query.q` → `reset()` → 自动重新加载搜索结果

## 四、数据模型

```python
# 用户资料
class UserProfile(models.Model):
    user = OneToOneField(User)
    photo = ImageField(upload_to='avatars/')
    bio = TextField()

# 面经笔记
class InterviewNote(models.Model):
    user = ForeignKey(User, related_name='interview_notes')
    title = CharField(200)
    content = TextField()
    cover = ImageField(upload_to='notes/covers/')
    company = CharField(100)
    position = CharField(100)
    difficulty = CharField(20, choices=['简单','中等','困难'])
    likes = PositiveIntegerField(default=0)

# 点赞（唯一约束）
class InterviewNoteLike(models.Model):
    user = ForeignKey(User)
    note = ForeignKey(InterviewNote)
    Meta.constraints = [UniqueConstraint(user, note)]

# 评论
class InterviewNoteComment(models.Model):
    user = ForeignKey(User)
    note = ForeignKey(InterviewNote, related_name='comments')
    content = TextField()
```

## 五、搜索功能数据流

```
NavBar: 输入"字节" → handleSearch()
  → router.push({ name:'home', query:{ q:'字节' } })
  → URL: /?q=字节

FeedWaterfall: watch(route.query.q) → reset()
  → GET /api/notes/?search_query=字节

get_list.py:
  search_query = request.GET.get('search_query')
  Q(title__icontains) | Q(content__icontains) |
  Q(company__icontains) | Q(position__icontains)
  → 返回匹配列表
```

## 六、API 部署与配置

### 6.1 开发环境
| 端口 | 服务 | 说明 |
|------|------|------|
| `:5173` | Vite dev server | HMR 热更新，`strictPort: true` |
| `:8000` | Django runserver | 后端 API + 托管前端 |

### 6.2 生产模式
```bash
cd frontend && npm run build   # → backend/static/frontend/
cd ../backend && python manage.py runserver
# 访问 localhost:8000，Django 同时服务前后端
```

### 6.3 关键配置

`vite.config.js`：
```js
server: { port: 5173, strictPort: true }
build: { outDir: '../backend/static/frontend', emptyOutDir: true }
```

`settings.py`：
```python
TEMPLATES = [{ 'DIRS': [BASE_DIR / 'static/frontend'] }]
MEDIA_URL = '/media/'
CORS_ALLOWED_ORIGINS = ['http://localhost:5173', 'http://127.0.0.1:5173']
```

`urls.py`：
```python
path('', index, name='index')
re_path(r'^(?!media/|static/|assets/).*$', index, name='frontend_fallback')
```

`api.js`：
```javascript
const BASE_URL = import.meta.env.DEV ? 'http://127.0.0.1:8000' : ''  // 生产同源
```
