## 一、项目概述

AIFriends 是一个 AI 面经社区平台，前后端分离架构：
- **后端**：Django 6.0.5（运行在 `http://127.0.0.1:8000`）
- **前端**：Vue 3 + Vite + TailwindCSS 4 + daisyUI 5（运行在 `http://localhost:5173`）
- 前后端域名不同，已配置 CORS 跨域支持
- 用户认证：JWT（`djangorestframework-simplejwt`），access_token 2h + refresh_token 7d
- 页面风格：小红书 Web 首页风格
- 核心功能：用户注册登录、发布面经、浏览首页瀑布流、点赞评论、个人资料管理

### 7.3 路由守卫逻辑

```javascript
router.beforeEach((to) => {
  const userStore = useUserStore()

  // 需要登录 且 已拉取用户信息 且 未登录 → 跳登录页（带 redirect）
  if (to.meta.requiresAuth && userStore.hasPulledUserInfo && !userStore.isLoggedIn) {
    return { name: 'login', query: { redirect: to.fullPath } }
  }

  return true
})
```

App.vue 的 `onMounted` 中在拉取用户信息后再次检查是否需要跳转（用 `router.replace` 防止后退）：

```javascript
onMounted(async () => {
  await userStore.pullUserInfo()

  const currentRoute = router.currentRoute.value
  if (currentRoute.meta.requiresAuth && !userStore.isLoggedIn) {
    router.replace({ name: 'login', query: { redirect: currentRoute.fullPath } })
  }
})
```

## 八、前端核心模块

### 8.1 HTTP 请求层（frontend/src/js/http/api.js）

```javascript
// 开发环境请求 127.0.0.1:8000，生产环境使用相对路径（同源）
const BASE_URL = import.meta.env.DEV ? 'http://127.0.0.1:8000' : ''

const api = axios.create({
  baseURL: BASE_URL,
  withCredentials: true,     // 携带 cookie（refresh_token）
  timeout: 5000,
})
```

**请求拦截器**：自动在 Authorization 头添加 `Bearer {access_token}`

**响应拦截器（401 自动刷新）**：
1. 收到 401 → 检查是否是 refresh_token 请求本身
   - 是 → 清除登录态
   - 否 → 进入刷新流程
2. 订阅者队列机制：并发 401 请求只发一次刷新请求
3. 刷新成功 → 更新 access_token → 重放所有等待的请求
4. 刷新失败 → 清除登录态 → 通知所有等待请求失败

### 8.2 Pinia 状态管理（frontend/src/stores/user.js）

```javascript
state: () => ({
  accessToken: localStorage.getItem('access_token') || '',
  user: loadStoredUser(),                    // 从 localStorage 恢复
  hasPulledUserInfo: false,                  // 是否已拉取用户信息
})

getters: {
  isLoggedIn: (state) => Boolean(state.accessToken && state.user),
}

actions: {
  setAuth(accessToken, user)  // 设置认证信息 + 持久化到 localStorage
  clearAuth()                 // 清除认证信息
  login(payload)              // POST /api/user/account/login/
  register(payload)           // POST /api/user/account/register/
  pullUserInfo()              // GET /api/user/account/info/（首屏调用）
  updateProfile(payload)      // POST /api/user/account/update_profile/
  refreshToken()              // POST /api/user/account/refresh_token/
  logout()                    // POST /api/user/account/logout/
}
```

关键设计：
- `access_token` 存 localStorage，`refresh_token` 存 httpOnly cookie
- `hasPulledUserInfo` 防止重复拉取
- 登录/注册成功后自动调用 `handleAuthResponse` 统一处理
- `logout()` 使用 `finally` 确保即使请求失败也清除本地状态

## 十二、常用命令速查

### 后端（在 `backend/` 目录下执行）

```bash
python manage.py runserver          # 启动开发服务器
python manage.py makemigrations     # 创建新的数据库迁移文件
python manage.py migrate            # 执行数据库迁移
python manage.py createsuperuser    # 创建超级管理员
python manage.py check              # Django 系统检查
python manage.py shell              # 进入 Django Shell
```

### 前端（在 `frontend/` 目录下执行）

```bash
npm run dev          # 启动开发服务器（HMR，固定 :5173）
npm run build        # 打包到 Django static 目录
npm install          # 安装依赖
```

### 运行服务（同时启动前后端）

| 终端 | 目录 | 命令 |
|------|------|------|
| 终端 1 | `backend/` | `python manage.py runserver` |
| 终端 2 | `frontend/` | `npm run dev` |

### 生产模式（Django 托管前端）

```bash
cd frontend && npm run build        # 打包前端
cd ../backend && python manage.py runserver  # 启动 → 访问 localhost:8000
```
