## 一、项目概述

AIFriends 是一个 AI 面经社区平台，前后端分离架构：
- **后端**：Django 6.0.5（运行在 `http://127.0.0.1:8000`）
- **前端**：Vue 3 + Vite + TailwindCSS v4 + daisyUI v5（运行在 `http://localhost:5173`）
- 前后端域名不同，已配置 CORS 跨域支持
- 用户认证：JWT（`djangorestframework-simplejwt`），access_token 2h + refresh_token 7d
- 页面风格：daisyUI 组件驱动 + Tailwind utility class
- 核心功能：用户注册登录、发布面经、搜索浏览、点赞评论、收藏管理、个人资料编辑
- 图片存储：全部以 Base64 字符串存入数据库 TextField，不依赖文件系统

## 四、数据模型

```python
# 用户资料
class UserProfile(models.Model):
    user = OneToOneField(User)
    photo_base64 = TextField(blank=True, default='')  # data:image/...;base64,...
    bio = TextField()

# 面经笔记
class InterviewNote(models.Model):
    user = ForeignKey(User, related_name='interview_notes')
    title = CharField(200)
    content = TextField()
    cover_base64 = TextField(blank=True, default='')  # data:image/...;base64,...
    company = CharField(100)
    position = CharField(100)
    difficulty = CharField(20, choices=['简单','中等','困难'])
    likes = PositiveIntegerField(default=0)

# 点赞（唯一约束）
class InterviewNoteLike(models.Model):
    user = ForeignKey(User)
    note = ForeignKey(InterviewNote)
    Meta.constraints = [UniqueConstraint(user, note)]

# 收藏（唯一约束）
class InterviewNoteFavorite(models.Model):
    user = ForeignKey(User, related_name='favorite_notes')
    note = ForeignKey(InterviewNote, related_name='favorite_records')
    created_at = DateTimeField(auto_now_add=True)
    Meta.constraints = [UniqueConstraint(user, note)]

# 评论
class InterviewNoteComment(models.Model):
    user = ForeignKey(User)
    note = ForeignKey(InterviewNote, related_name='comments')
    content = TextField()
```

## 六、图片存储方案（Base64 + TextField）

### 6.1 为什么选 Base64

- **自包含**：数据库导出即包含所有图片，不依赖 media/ 文件夹
- **迁移简单**：只需搬运一个 `db.sqlite3` 文件
- **部署无麻烦**：不需要处理文件上传权限、媒体文件路由

### 6.2 数据流

```
前端: input[type=file] → FileReader.readAsDataURL()
  → data:image/png;base64,... → POST JSON { cover_base64: "data:..." }

后端 image_utils.py:
  process_base64(raw_b64):
    base64.b64decode() → 二进制
    if > 2MB → Image.thumbnail(1200,1200) → JPEG quality 85
    base64.b64encode() → data:image/jpeg;base64,...
  → 存到 TextField

序列化:
  serialize_note(): note.cover_base64 → { "image": "data:..." }
```

### 6.3 image_utils.py 核心函数

```
process_base64(raw_b64) → str      # 解码→压缩→重编码→data URI
_resize(raw_bytes) → bytes          # PIL thumbnail + JPEG 85%
_guess_format(raw_bytes) → str     # 用 PIL 替代已废弃的 imghdr
```

## 七、收藏功能

### 7.1 数据流

```
卡片/详情页: 点击 ★ 按钮
  → POST /api/notes/<id>/toggle_favorite/
  → InterviewNoteFavorite.objects.get_or_create(user, note)
  → { favorited: true } or { favorited: false }

收藏列表页: /favorites/（requiresAuth）
  → GET /api/notes/favorites/
  → InterviewNoteFavorite.objects.filter(user=request.user).order_by('-created_at')
  → 遍历 fav.note → serialize_note() → 列表渲染
```

### 7.2 前端响应

- 序列化时自动包含 `favorited: bool`
- 卡片底部：收藏按钮（☆/★）+ 点赞（♥）并列

- 侧栏"我的收藏"路由从 `/profile/` 改为 `/favorites/`

## 八、阿里云 TTS 语音合成

### 8.1 为什么替换浏览器 SpeechSynthesis

浏览器原生 `SpeechSynthesis` API 在移动端（尤其 iOS）存在多个问题：
- **需手动唤醒**：iOS 上首次调用 `speak()` 前必须播放一段空语音，否则静默失败
- **音色生硬**：系统 TTS 语音机械感强，口语场景听感差（AI 味道浓）
- **语音列表不可靠**：`onvoiceschanged` 在 iOS 上经常不触发
- **手势限制**：异步回调中的 `speak()` 可能被移动端浏览器拦截

### 8.2 架构

```
用户点击"朗读" / 面试模式自动播放
  → FriendIndex.vue speak(index, text)
    → POST /api/tts/synthesize/  { text, voice }
      → web.utils.aliyun_tts.synthesize()
        → _get_token()  [缓存 Token]
          → POST https://nls-meta.cn-shanghai.aliyuncs.com/api/v1/token
        → POST https://nls-gateway.aliyuncs.com/stream/v1/tts  [返回 MP3 二进制]
      → base64 编码后返回前端
    → Audio 元素播放 base64 MP3
```

### 8.3 音色配置

| 性别 | 音色名 | 类型 | 说明 |
|------|--------|------|------|
| 女声 | `ailun`（默认） | 精品 | 温暖自然，适合 AI 助手 |
| 男声 | `aicheng`（默认） | 精品 | 沉稳成熟，适合面试官 |
| 女声 | `xiaoyun` | 标准 | 阿里云免费音色 |
| 男声 | `xiaogang` | 标准 | 阿里云免费音色 |

> 标准音色免费使用，精品音色需付费。在 `aliyun_tts.py` 的 `VOICE_MAP` 中可切换。

### 8.4 Token 管理

- Token 从阿里云 NLS 获取，有效期通常 24 小时
- 模块内使用进程级内存缓存（`TOKEN_CACHE` 字典），提前 5 分钟刷新
- 每次语音合成请求自动检查缓存，过期后重新获取

### 8.5 前置条件

```bash
# backend/.env 中配置
ALIYUN_NLS_ACCESS_KEY_ID=你的AccessKeyId
ALIYUN_NLS_ACCESS_KEY_SECRET=你的AccessKeySecret
ALIYUN_NLS_APPKEY=你的项目AppKey
```

开通步骤：
1. 登录阿里云控制台 → 智能语音交互（NLS）
2. 创建项目 → 获取 AppKey
3. 创建 RAM 用户 → 授权 AliyunNLSServiceRole → 获取 AccessKey

## 十、生产部署架构

